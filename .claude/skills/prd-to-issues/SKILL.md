# PRDからイシューへ

PRDを垂直スライス（tracer-bullet）を用いて、個別に割り当て可能なGitHubイシューに分割する。

## 用語定義

- **tracer-bullet (曳光弾)**: システム全体を横断する「薄い垂直スライス（Vertical Slice）」の実装手法。特定のレイヤーを水平に作るのではなく、スキーマ、API、UI、テストといった全レイヤーを端から端まで貫通する形で、検証可能な最小単位として実装することを指します。
- **HITL (Human In The Loop)**: 人の介入が必要なスライス。アーキテクチャの決定、設計レビュー、重要な技術的判断など、人間の判断や承認が必要な作業項目を指します。
- **AFK (Away From Keyboard)**: 人の介入なしに実装およびマージできるスライス。明確に定義された要件に基づいて、自動化されたプロセスや独立した実装が可能な作業項目を指します。

## Process

### 1. Locate the PRD

ユーザーにPRDのGitHubイシュー番号（または URL）を尋ねます。

PRD がコンテキストウィンドウにまだ表示されていない場合は、`gh issue view <番号>`（コメント付き）で取得します。

### 2. コードベースを確認する（任意）

まだコードベースを調査していない場合は、コードの現状を把握するために調査を行ってください。

### 3. 垂直スライスの草案作成

PRDを**tracer bullet**のイシューに分割します。各イシューは、単一のレイヤーの水平スライスではなく、すべての統合レイヤーをエンドツーエンドで貫く薄い垂直スライスです。

スライスは「HITL」または「AFK」のいずれかになります。HITLスライスには、アーキテクチャの決定や設計レビューなど、人の介入が必要です。AFKスライスは、人の介入なしに実装およびマージできます。可能な限り、HITLよりもAFKを優先してください。

<vertical-slice-rules>
- 各スライスは、すべてのレイヤー（スキーマ、API、UI、テスト）を網羅する、範囲は狭いながらも完全なパスを提供します
- 完成したスライスは、単独でデモや検証が可能です
- 少数の分厚いスライスよりも、多数の薄いスライスを優先します
</vertical-slice-rules>

### 4. ユーザーにクイズ

提案された分割内容を番号付きリストとして提示してください。各スライスについて、以下を示してください：

- **タイトル**：簡潔な説明名
- **タイプ**：HITL / AFK
- **ブロック元**：先に完了する必要がある他のスライス（ある場合）
- **対象となるユーザーストーリー**：PRD（製品要件仕様書）のうち、どのユーザーストーリーに対応するか

ユーザーに確認してください：

- 細分化の粒度は適切ですか？（粗すぎる／細かすぎる）
- 依存関係は正しいですか？
- スライスを統合したり、さらに分割したりすべきですか？
- 正しいスライスが HITL および AFK としてマークされていますか？

ユーザーがこの内訳を承認するまで、このプロセスを繰り返してください。

### 5. Create the GitHub issues

承認された各スライスについて、`gh issue create` コマンドを使用して GitHub のイシューを作成してください。イシュー本文には、以下のテンプレートを使用してください。

「Blocked by」フィールドに実際のイシュー番号を記載できるよう、依存関係順（ブロック要因となるものを優先）でイシューを作成してください。

<issue-template>
## Parent PRD

#[prd-issue-number]

## What to build

この垂直スライスの概要を簡潔に記述してください。レイヤーごとの実装ではなく、エンドツーエンドの動作について説明してください。内容を重複させないよう、親PRDの具体的なセクションを参照してください。

## 受け入れ基準

- [ ] 基準 1
- [ ] 基準 2
- [ ] 基準 3

## Blocked by

- Blocked by #<issue-number> (if any)

または、ブロックがない場合、 "None - can start immediately" としてください。

## 対象となるユーザーストーリー

親PRDからの番号による参照：

- ユーザーストーリー 3
- ユーザーストーリー 7

</issue-template>

親のPRD課題を閉じたり、変更したりしないでください。
